## 2026-06-22  Phase 6  Build Layer 6 calibration, RPS and backtest
- Did: Implemented calibrate.py, the honesty layer that scores the forecast against real results, with pure scoring separated from backtest orchestration. The metric is the Ranked Probability Score, the right score for ordered three-outcome football, lower is better. rps and mean_rps wrap penaltyblog's rps_array and rps_average, confirmed against the installed 1.11.0 API, rather than hand-rolling. outcome_index and base_rates are pure too, so the gate tests the scorer with no model and no network. run_backtest scores out of sample under strict leakage discipline: it takes the most recent backtest_size matches as a holdout window, fits the goal model once on data strictly before the window start, and asserts training max date is strictly before the holdout start, treating any overlap as a failure. Each holdout match is predicted honouring its actual venue and scored with RPS. A single training cutoff keeps it to one fit because fitting per historical match is expensive, the window is recent and short so the fit is only slightly stale, a conservative bias, and backtest_size is configurable, default 150. The naive baseline is the training base rates applied as a constant on the same matches, computed from training only so both sides are leakage-safe. Added a calibrate block to config.yaml, a wc-calibrate entry point in pipeline.py that prints and logs the model and baseline mean RPS, the matches scored, and the cutoff dates, leaving the forecast flow intact, and artifact.write_calibration_log reusing the provenance pattern. ## 2026-06-23  Phase 7.8  Hybrid ensemble, measured against Dixon-Coles
- Did: Built the hybrid ensemble, a separate per-match win, draw, loss model competing with Dixon-Coles on the same metric and the exact Phase 7.6 walk-forward splits, without touching the Dixon-Coles scoreline model. Model is scikit-learn HistGradientBoostingClassifier, the standard library gradient boosting, picked for deterministic seeded reproducibility and native missing-feature handling with no native-binary friction. Four leakage-safe home-minus-away differential features: point-in-time Dixon-Coles net strength, squad value, point-in-time Elo, rest-days. Evaluation reuses calibrate.walk_forward_windows and recomputes Dixon-Coles and baseline per window via score_window, reproducing 0.1575. Added wc-ensemble, artifact.write_ensemble_log, the holdout index on BacktestWindow, and tests/test_ensemble_gate.py.
- Point-in-time ability enforcement: a grid of refit dates at a fixed cadence, default 120 days; at each grid date Dixon-Coles is fit only on data strictly before that date within a rolling lookback; a match takes the abilities from the latest grid date at or before it, so its ability uses only data strictly before its own date.
