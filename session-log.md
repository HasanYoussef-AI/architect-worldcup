## 2026-06-23  Walk-forward backtest, the measurement instrument and noise floor
- Did: replaced the single 150-match holdout with a rolling-origin walk-forward backtest, the instrument every later change is judged against. It reuses the trusted machinery rather than forking it: extracted score_window from run_backtest (so run_backtest is unchanged and still 0.1611), and every window is built by prepare_backtest and guarded by assert_no_leakage. run_walk_forward anchors the most recent window on the calibration window, then steps backward in consecutive non-overlapping 150-match holdouts, each window's as_of set to the previous window's train_max_date so the holdouts are contiguous and share no matches. The artifact records per-window results, aggregate statistics, the per-window leakage cutoffs, and the explicit window definitions including holdout match identifiers, so later phases reuse identical splits.
- Result: 8 non-overlapping windows. Per-window model RPS ranged 0.1336 to 0.1764; the model beat its own baseline in all 8. Aggregate mean 0.1575, standard deviation 0.0140, mean baseline 0.2292; the anchor window reproduced the frozen 0.1611 to the digit.
- Noise floor, the standing rule: the across-window standard deviation is 0.0140. No future change counts as real unless it moves the aggregate mean RPS by more than that. For context, the single-window squad delta (0.0022) and the friendly downweight (0.0006) both sit well inside this floor, so neither is distinguishable from window-to-window variance on this evidence.

## 2026-06-23  Hybrid ensemble, measured and rejected
- Did: built a hybrid ensemble, a separate per-match win/draw/loss model competing with Dixon-Coles on the same metric and the exact walk-forward splits, without touching the Dixon-Coles scoreline model. The model is scikit-learn HistGradientBoostingClassifier, chosen for deterministic seeded reproducibility and native missing-feature handling. Four leakage-safe home-minus-away differential features: point-in-time Dixon-Coles net strength, squad value, point-in-time Elo, rest-days. Point-in-time abilities are enforced by a refit-date grid: at each grid date Dixon-Coles is fit only on data strictly before it, and a match takes its abilities from the latest grid date at or before it, so its features use only data strictly before its own date; the gate proves that tampering a match's own and every later outcome leaves its features unchanged.
- Deliberately excluded, by design: bookmaker odds, so the model can be benchmarked against the market rather than copying it; and socio-economic features (GDP, population) as near-noise.

## 2026-06-25  LLM research phase, allow-listed web search and filter-and-record gates
- Did: wired the research phase. ## 2026-07-02  Prediction A is not invariant across as-of, a read-only finding
- Did: an offline pre-flight for match 83 found Prediction A is deterministic at a fixed as-of, bit-reproducible, but not bit-invariant across as-of on the frozen snapshot. The scored training rows are identical across the two dates, so the only differing input is base_date, which moves the weight scale and, through the optimizer, nudges the fit. An earlier pre-flight guess, that unscored rows were entering the fit, was wrong and is corrected here by reading the code.
- Decision: fit_model is shared across five call sites, so pinning base_date to the max scored date would make A invariant but is a global change that would move the frozen calibration numbers. That is a design decision, not a scoped fix, and forcing invariance would break the intended recency weighting. No change made; each live run pins a single canonical as-of, the day of kickoff, so each committed A is reproducible at that as-of.
