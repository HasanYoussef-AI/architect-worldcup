## What this project is

A live World Cup 2026 bracket re-simulator with a verification-first architecture. It is a portfolio project and build-in-public content for The Architect AI. It runs from the command line on manual trigger. There is no web app, no Docker, and no scheduler.

## Architecture (the output contract is the spine)

The model layer produces structured JSON and nothing else. Every downstream consumer reads that JSON. Never entangle model logic with presentation. Modules map to layers:

- ingest.py    Layer 1  data integrity, dated snapshots, leakage guard
- ratings.py   Layer 2  Elo
- model.py     Layer 3  Dixon-Coles via penaltyblog
- squad.py     Layer 4  squad-value adjustment, dated snapshot
- simulate.py  Layer 5  Monte Carlo bracket
- calibrate.py Layer 6  RPS, backtest
- ablation.py  Layer 7  with and without harness
- artifact.py  versioned JSON plus run log (provenance: config, as_of_date, seed, git sha, UTC time)
- pipeline.py  the single entry point

## Build principles

- Walking skeleton first, then deepen. Keep the pipeline runnable end to end at every phase.
- Reproducibility is non-negotiable. Fixed seed, immutable dated raw snapshots, provenance log on every run. Same seed and same data must produce the same output.
- Verification gates are real tests, not prose. A no-leakage check, schema validation, reproducibility, and calibration are pytest tests that must pass. If a gate cannot be expressed as a test, it is not a gate.
- Do not chase model accuracy and do not add a UI. The architecture and the honesty are the product.
