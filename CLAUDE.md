# CLAUDE.md, Architect WorldCup

This file governs how Claude Code works in this repository. Read it at the start of every session and follow it.

## What this project is

A live World Cup 2026 bracket re-simulator with a verification-first architecture. It is a portfolio project and build-in-public content for The Architect AI. It runs from the command line on manual trigger. There is no web app, no Docker, and no scheduler.

## Session protocol (mandatory)

At the start of every session:
1. Read this file.
2. Read session-log.md and continue from the last "Next" line.
3. State where you are resuming and the next action before writing any code.

After every commit:
1. Append its session-log.md entry in the same step that makes the commit. A commit is not done until its entry is written. This is how future sessions continue cleanly, so never skip it.
2. A batch of related commits, an audit or a refactor sweep, gets one narrative entry that cites each hash, not one entry per trivial commit.

session-log.md entry format:
```
## YYYY-MM-DD  short descriptive title
- Did: what changed this session and why, in reader-facing prose
- Commit: <sha> <message>   (one line per commit; a batch lists each)
- Verified: gates run and result
- Next: the single next action
```
Add labeled lines as the entry needs them, for example a finding, a decision and its reasoning, or a measured result.

The session-log is public. Keep every entry reader-facing:
- No cost or token figures. A relative result, such as a percentage change, is fine.
- No tooling internals, harness mechanics, or off-repo working notes.
- No personal-machine incidents. State a security practice as a practice, not as plumbing.
- No em dashes, no emojis.

## Architecture (the output contract is the spine)

The model layer produces structured JSON and nothing else. Every downstream consumer reads that JSON. Never entangle model logic with presentation. Modules map to layers:

- ingest.py    Layer 1  data integrity, dated snapshots, leakage guard
- ratings.py   Layer 2  Elo
- model.py     Layer 3  Dixon-Coles via penaltyblog
- squad.py     Layer 4  squad-value adjustment, dated snapshot
- simulate.py  Layer 5  Monte Carlo bracket
- calibrate.py Layer 6  RPS, backtest
- ablation.py  Layer 7  with and without harness
- artifact.py  versioned JSON plus run log (provenance: config, as_of_date, seed, git sha, UTC time)
- pipeline.py  the single entry point

## Build principles

- Walking skeleton first, then deepen. Keep the pipeline runnable end to end at every phase.
- Reproducibility is non-negotiable. Fixed seed, immutable dated raw snapshots, provenance log on every run. Same seed and same data must produce the same output.
- Verification gates are real tests, not prose. A no-leakage check, schema validation, reproducibility, and calibration are pytest tests that must pass. If a gate cannot be expressed as a test, it is not a gate.
- Do not chase model accuracy and do not add a UI. The architecture and the honesty are the product.

## Style rules (all output, including code comments and commit messages)
